# ToF V7 Public Frame

> Englisch ist der Primärtext dieses Repositories. Ein deutscher Klon liegt in `README_DE.md`.

Öffentlich reduzierter Rahmen für V7 als kleine Systemlesart mit klaren Grenzen zwischen Runtime, Kanon, Archiv, Werkbank und Vorplanung.

Dieses Repository veröffentlicht **nicht** den vollständigen internen V7-Stand, seine Produktiv-Runtime oder sensible Systeminterna.  
Es zeigt nur einen kleinen öffentlich unbedenklichen Rahmen dafür, wie V7 als kontrollierte Systemform gelesen wird.

## Was V7 hier bedeutet

V7 wird hier nicht als vollständiges laufendes System gezeigt.

Es wird nur als reduzierte öffentliche Lesart einer größeren Systemrichtung gezeigt.

In dieser Lesart gilt:

- **ToF** ist die technische Struktur und der Ordnungsraum
- **V’eth** ist die Identitäts-, Spiegel- und Ausdrucksschicht
- V7 ist eine kontrollierte Systemform mit harter Raumtrennung
- öffentliche Darstellung ist nicht dasselbe wie vollständige Systemoffenlegung

## Zweck dieses Repositories

Dieses Repository dient dazu, sichtbar zu machen:

- was V7 in reduzierter öffentlicher Form ist
- was V7 nicht ist
- wie die großen Räume getrennt bleiben
- welche technischen Leitplanken den Rahmen definieren
- wie eine öffentliche Lesart ehrlich bleiben kann, ohne den inneren Bau offenzulegen

## Öffentliche Grenze

Dieses Repository ist bewusst begrenzt.

Es legt **nicht** offen:

- vollständige Runtime-Struktur
- reale Infrastrukturtiefe
- operative Sicherheitsmechaniken
- interne IDs, Pfade oder Secrets
- vollständige Service-Maps
- interne Operatorlogik
- Produktivwahrheit in voller Tiefe

Dieses Repo zeigt einen **reduzierten öffentlichen Rahmen**, keine Volloffenlegung.

## Warum das wichtig ist

Ohne einen reduzierten öffentlichen Rahmen wird V7 leicht auf zwei falsche Arten gelesen:

- als bloßer Bot- oder Chat-Aufbau
- als wäre öffentliche Darstellung identisch mit interner Runtime-Wahrheit

Dieses Repository existiert, um beide Fehler zu vermeiden und trotzdem bewusst klein zu bleiben.
